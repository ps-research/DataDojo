# DataDojo

An online judge for data skills. Users solve analytical problems in five SQL
dialects (SQLite, DuckDB, PostgreSQL, MySQL, SQL Server), Python (pandas), and R,
and get automatic verdicts (AC / WA / TLE / RE / CE) against hidden test data.

Live: https://datadojo.duckdns.org

## Why

Practice for data roles is scarce and paywalled. LeetCode locks most of its
database problems. DataDojo is an open, curriculum-driven set of SQL and
data-analysis problems.

## Features

- 89 problems across five belts (White to Red), grouped into themed universes
  (e-commerce, streaming, ride-hailing, hospital ops, SaaS analytics, an
  exchange order book) plus tutorials and dedicated Python/R problems.
- Seven judge engines. The same SQL problem is playable in any of the five SQL
  dialects; Python and R have their own problems.
- Run against a visible sample to see your output, or Submit to judge against
  three hidden test sets (different data, so you cannot hardcode the sample).
- Monaco editor (the VS Code editor) with per-language syntax and autocomplete.
- AI hints: an assistant reads the problem, the reference solution, and your
  code, and gives one substantial hint without solving it. Free-tier Gemini keys
  rotated on quota; ten hints per user per hour.
- JWT auth with refresh tokens, trusted-provider email signup, optional email
  OTP verification, per-user profile stats, settings, and a global leaderboard.
- Light and dark themes, resizable panels, focus mode.

## Stack

React + TypeScript (Vite, Tailwind, Monaco) - Express + TypeScript - MongoDB
(Mongoose) - Redis (cache, leaderboard, rate limits, BullMQ queue) - nginx with
Let's Encrypt TLS. Judge engines: SQLite and DuckDB in-process, PostgreSQL,
MySQL/MariaDB, and SQL Server as servers, Python and R as sandboxed subprocesses.

## Content provenance

Problems derive from the *SQL Cookbook, 2nd Edition* (Molinaro and de Graaf,
O'Reilly) as the skill spine, with the official PostgreSQL documentation as the
reference. A pipeline under `kb/` extracts the book verbatim (sha256-tracked),
parses recipes and their dialect solutions, and executes every solution on all
five SQL engines before anything is used. Problem statements are original;
datasets are generated by seeded scripts. See `docs/CONTENT-SPEC.md`.

## Local development

```
# API (needs MongoDB and Redis running locally)
cd app/api && npm install && cp .env.example .env && npm run dev

# Web
cd app/web && npm install && npm run dev
```

The web dev server proxies `/api` to the backend on port 4000.

## Deployment

Single server (Ubuntu 24.04). `deploy/provision.sh` installs Node, the databases,
Python, R, nginx, and certbot. `deploy/deploy.sh` builds both apps, serves the
SPA through nginx, and runs the API under systemd. Content is built and seeded
with `content/build_gold.py` then `npm run seed`.
